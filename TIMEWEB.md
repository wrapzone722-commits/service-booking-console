# Настройка приложения в Timeweb Cloud

Если в логах видно **«port 3000»** и затем **«Received SIGTERM»** — платформа проверяет другой порт (обычно 8080), health check не проходит, контейнер завершается.

## Что сделать один раз

1. Откройте панель **Timeweb Cloud** → ваше приложение.
2. Перейдите в **Настройки** (или **Переменные окружения**).
3. Найдите переменную **`PORT`**:
   - Если **PORT=3000** — измените на **`8080`** или удалите переменную (тогда приложение само поднимется на 8080).
   - Если переменной нет — добавьте **`PORT`** = **`8080`**.
4. Сохраните настройки.
5. Запустите **«Пересобрать»** / **«Создать сборку»** (чтобы подтянуть последний код и новый порт).

После деплоя в логах должно быть: **«Fusion Starter server running on port 8080»**. Контейнер не должен получать SIGTERM — сайт начнёт открываться.

## Порт приложения в настройках Timeweb

В настройках приложения (порт контейнера / порт приложения) укажите **8080** — тот же, что и переменная `PORT`.

## Проверка состояния (health check)

Путь проверки: **`/health`**. Эндпоинт возвращает `200 OK`.
